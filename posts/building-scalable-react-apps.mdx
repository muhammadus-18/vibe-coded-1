# Building Scalable React Apps

As React applications grow in complexity, maintaining code quality and performance becomes increasingly challenging. This guide explores proven architectural patterns and best practices for building scalable React applications.

## Architecture Principles

### Single Responsibility Principle

Each component should have one reason to change:

```tsx
// Bad: Multiple responsibilities
const UserCard = ({ user }) => {
  const [isEditing, setIsEditing] = useState(false);
  const [formData, setFormData] = useState(user);
  
  const handleSave = async () => {
    await updateUser(user.id, formData);
    setIsEditing(false);
  };
  
  return (
    <div>
      {isEditing ? (
        <UserEditForm 
          formData={formData} 
          onChange={setFormData}
          onSave={handleSave}
        />
      ) : (
        <UserDisplay user={user} />
      )}
    </div>
  );
};

// Good: Single responsibilities
const UserCard = ({ user }) => {
  return (
    <div>
      <UserDisplay user={user} />
      <UserEditToggle />
    </div>
  );
};
```

### Composition Over Inheritance

Leverage React's composition model:

```tsx
// Flexible card component
const Card = ({ children, className, ...props }) => (
  <div className={`card ${className}`} {...props}>
    {children}
  </div>
);

const CardHeader = ({ children }) => (
  <div className="card-header">{children}</div>
);

const CardBody = ({ children }) => (
  <div className="card-body">{children}</div>
);

// Usage
<Card>
  <CardHeader>Title</CardHeader>
  <CardBody>Content</CardBody>
</Card>
```

## State Management Strategies

### When to Use State Management

Choose the right tool for your needs:

| Scale | Solution | Use Case |
|-------|----------|----------|
| Small | Local State | Component-specific state |
| Medium | Context API | Shared state across components |
| Large | Redux/Zustand | Complex state management |
| Enterprise | Micro-frontends | Multiple teams, large apps |

### Custom Hooks Pattern

Encapsulate business logic:

```tsx
// Custom hook for user data
const useUser = (userId) => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetchUser(userId)
      .then(setUser)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [userId]);

  const updateUser = useCallback(async (userData) => {
    try {
      const updated = await updateUserAPI(userId, userData);
      setUser(updated);
    } catch (err) {
      setError(err);
    }
  }, [userId]);

  return { user, loading, error, updateUser };
};
```

## Performance Optimization

### Code Splitting

Implement lazy loading for better performance:

```tsx
import { lazy, Suspense } from 'react';

const AdminPanel = lazy(() => import('./AdminPanel'));
const UserDashboard = lazy(() => import('./UserDashboard'));

function App() {
  return (
    <div>
      <Navigation />
      <Suspense fallback={<LoadingSpinner />}>
        <Routes>
          <Route path="/admin" element={<AdminPanel />} />
          <Route path="/dashboard" element={<UserDashboard />} />
        </Routes>
      </Suspense>
    </div>
  );
}
```

### Memoization Strategies

Prevent unnecessary re-renders:

```tsx
// Expensive calculation
const ExpensiveComponent = ({ data, filter }) => {
  const filteredData = useMemo(() => {
    return data.filter(item => item.category === filter);
  }, [data, filter]);

  return (
    <div>
      {filteredData.map(item => (
        <Item key={item.id} item={item} />
      ))}
    </div>
  );
};

// Event handler optimization
const Button = ({ onClick, children }) => {
  const handleClick = useCallback((e) => {
    onClick(e);
  }, [onClick]);

  return <button onClick={handleClick}>{children}</button>;
};
```

## Testing Strategies

### Testing Pyramid

Follow the testing pyramid for comprehensive coverage:

1. **Unit Tests** (70%): Test individual functions and components
2. **Integration Tests** (20%): Test component interactions
3. **E2E Tests** (10%): Test user workflows

### Component Testing

```tsx
// UserCard.test.tsx
import { render, screen, fireEvent } from '@testing-library/react';
import { UserCard } from './UserCard';

describe('UserCard', () => {
  const mockUser = {
    id: 1,
    name: 'John Doe',
    email: 'john@example.com',
  };

  test('renders user information correctly', () => {
    render(<UserCard user={mockUser} />);
    
    expect(screen.getByText('John Doe')).toBeInTheDocument();
    expect(screen.getByText('john@example.com')).toBeInTheDocument();
  });

  test('handles edit mode toggle', () => {
    render(<UserCard user={mockUser} />);
    
    const editButton = screen.getByRole('button', { name: /edit/i });
    fireEvent.click(editButton);
    
    expect(screen.getByDisplayValue('John Doe')).toBeInTheDocument();
  });
});
```

### Monitoring and Error Tracking

Implement proper monitoring:

```tsx
// Error boundary
class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false };
  }

  static getDerivedStateFromError(error) {
    return { hasError: true };
  }

  componentDidCatch(error, errorInfo) {
    // Send error to monitoring service
    trackError(error, errorInfo);
  }

  render() {
    if (this.state.hasError) {
      return <ErrorFallback />;
    }

    return this.props.children;
  }
}
```

## Conclusion

Building scalable React applications requires thoughtful architecture, proper state management, and performance optimization. By following these patterns and best practices, you can create applications that grow gracefully with your team and user base.

Remember that scalability is not just about handling more users; it's about maintaining code quality, developer productivity, and system reliability as your application evolves.
